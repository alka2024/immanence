英語スクールのオペレーション管理

講師の稼働・入金・生徒対応を一元管理。
現場の手作業を徹底的に減らし、講師・事務両方の業務負担を軽減しました。

開発の背景

講師の勤怠・入金確認・生徒の問い合わせ対応など、管理業務が分散しており非効率でした。 これらを一元化することで、少人数でも円滑に運営できる仕組みを目指しました。

主な機能

  • 講師の勤怠・稼働管理の自動化
    • レッスン開始・終了に連動した自動勤怠記録（打刻不要）
    • 講師ごとの稼働時間・コマ数・休講数を自動集計
    • 月次レポートを自動生成し、給与・報酬計算に活用可能
    • 代講依頼・承認フローも管理画面から完結
  • 生徒の入金管理（売上管理）
    • Stripe／銀行入金のステータスと生徒台帳を自動照合
    • 入金遅延者の自動検出と、メール／LINEでのリマインド通知
    • 過去入金履歴の一覧表示（担当講師も確認可能）
    • 決済エラー・未入金の自動フラグ付け
  • 問い合わせ・対応チケット管理
    • 生徒の問い合わせをチケット化し、未対応を自動検知
    • 指定時間以上返信がないチケットを担当講師へ自動通知
    • シフト終了前に「未対応一覧」を自動送信
    • 問い合わせ内容をカテゴリ別に集計し、改善ポイントを可視化
  • レッスン進捗・学習ログ管理
    • 担当講師用の進捗ダッシュボードで学習状況を一元確認
    • 課題提出・レッスン参加履歴を自動ログ化
    • 生徒・講師間で進捗ステータスをリアルタイム共有
  • SlackとのAPI連携によるリアルタイム運営
    • 講師の勤怠開始／終了をSlackに自動投稿
    • 生徒からの問い合わせ到着をリアルタイム通知
    • 入金確認の完了／未確認をチーム内で共有
    • 「今日のタスク一覧」「未対応チケット」を朝礼時間に自動配信
  • 事務・運営業務の自動化
    • 週次・月次レポートを自動生成し、管理者へ送信
    • 契約更新タイミングを検出し、生徒へ自動案内
    • 新規入会者情報をSlackやスプレッドシートへ自動同期
    • 退会処理（残コマ精算・担当講師通知）を半自動化
  • 三方連携ダッシュボード（講師 × 生徒 × 管理）
    • レッスン枠・稼働状況・担当生徒を一画面で管理
    • 生徒の入金状況・学習進捗・問い合わせ状況を統合管理
    • 管理者が全講師の稼働をリアルタイム把握可能
  • 低コスト運用のインフラ設計
    • GAS × Slack API × 外部決済APIでサーバーコストを最小化
    • 管理業務の自動化により、事務工数を30〜50%削減
    • 属人性を排除し、仕組みによる運営体制を実現

使用技術

  • Google Apps Script（GAS）
  • Slack Webhook API
  • Google Spreadsheet + Drive Integration

導入効果

勤怠・請求・問い合わせ対応が自動化され、事務工数を60%以上削減。 また、Slack通知により講師の対応漏れがゼロになりました。
